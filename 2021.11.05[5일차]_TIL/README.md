## 함수
### 함수란?
함수는 JS에서 가장 중요한 핵심 개념이다.  
다른 JS핵심 개념인 스코프, 실행 컨텍스트, 클로저, 생성자 함수에 의한 객체 생성, 메서드, this, 프로토타입, 모듈화 등이 모두 함수와 깊은 관련이 있다.  
수학의 함수는 입력을 받아 출력을 내보내는 일련의 과정이다. 프로그래밍 언어도 수학의 함수와 같은 개념이다.  
프로그래밍 언어의 함수는 일련의 과정을 문으로 구현하고 코드 블록으로 감싸서 하나의 실행단위로 정의한것이다.  

함수내부로 입력을 받는 변수를 매개변수, 입력을 인수, 출력을 반환이라고 한다.  
함수는 함수 정의를 통해 생성된다. 하지만 정의만으로는 함수가 생성되지 않는다.  
안수를 매개변수를 통해 함수에 전달하면서 함수의 실행을 명시적으로 지시해야하고 이를 함수 호출이라고 한다.  
함수를 호출하면 코드 블록에 담긴 문들이 일괄적으로 실행 되고 실행결과를 반환한다. (반환값)

### 함수를 사용하는 이유
함수는 필요할때 여러번 호출가능하고 실행시점을 개발자가 결정한다.  
그리고 몇번이든 재사용가능하기 때문에 코드의 재사용 측면에서 매우 유용하다.  
코드의 중복을 없애 재사용성을 높이는 함수는 유지보수의 편의성을 높이고 실수를 줄여 코드의 신뢰성을 확보한다.

함수는 객체 타입의 값이다. 따라서 이름 (식별자)를 붙일 수 있다.  
적절한 함수의 이름은 내부 코드를 이해하지 않아도 함수의 역할을 이해 할 수 있게 해준다.  
코드는 동작하는것 만이 존재 목적이 아니다. 코드는 개발자를 위한 문서이므로 사람이 이해 할 수 있도록 가독성이 좋게 만들어야한다.  

### 함수 리터럴
함수도 함수 리터럴을 통해 생성이 가능하다.  
함수리터럴은 function키워드, 함수 이름, 매개변수목록, 함수 몸체로 구성된다.  
```js
var f = function add (x,y){
  return x+y;
};
```

#### 함수 리터럴의  구성요소
구성요소 | 설명
--|--
함수이름 | 함수이름은 식별자다. 따라서 식별자 네이밍 규칙을 준수 해야한다. </br> 함수 이름은 함수 몸체 내에서만 참조 할 수 있는 식별자이다.  </br> 함수이름은 생략할 수 있다. 이름이 있는 함수를 기명함수, 이름이 없는함수를 무명/익명함수라고한다.
매개변수 목록 | 0개 이상의 매개변수를 소괄호로 감싸고 쉼표로 구분한다. </br>각 매개변수에는 함수를 호출할 때 지정한 인수가 순서대로 할당된다. 즉, 매개변수 목록은 순서에 의미가 있다.  </br> 매개변수는 함수 몸체 내에서 변수와 동일하게 취급된다. 따라서 매개변수도 변수와 마찬가지로 식별자 네이밍 규칙을 준수해야한다.
함수 몸체 | 함수가 호출되었을떄 일괄적으로 실행될 문들을 하나의 실행단위로 정의한 코드블록이다.  </br>함수몸체는  함수 호출에 의해 실행된다

리터럴은 값을 생성하기 위한 표기법이다. 함수 리터럴도 평가되어 값을 생성하며 이는 객체이므로 함수는 객체이다.  
함수는 객체지만 일반 객체와는 다르다. 일반 객체는 호출이 불가능하지만 함수는 호출할 수 있고 고유의 프로퍼티를 갖는다.  
함수가 객체라는 사실은 다른 프로그래밍 언어와 다른 자바스크립트의 큰 특징이다.

### 함수 정의 
함수 정의란 함수를 호출하기 이전에 인수를 전달받은 매개변수와 실행할 문들, 그리고 반환할 값을 지정하는것이다.  

함수 정의 방법
 - 함수선언문
 - 험수 표현식
 - function 생성자 함수
 - 화살표함수(ES6)

위의 정의방법은 함수를 생성하는 기능은 갖는다. 단, 미묘하지만 중요한 차이를 갖는다.
#### 함수선언문
  함수선언문은 함수리터럴과 형태가 동일하다.  
  단, 함수 리터럴은 함수이름을 생략가능하다 함수선언문은 함수이름을 생략 할 수 없다.  
  함수 선언문은 표현식이 아닌 문 따라서 완료값은 undefined이다.  
  표현식이 아닌문은 변수에 할당할 수 없으므로 함수 선언문도 변수에 할당할 수 없다.  
  하지만 다음 예제를 보면 함수 선언문을 변수에 할당하는것 처럼 보인다.  
```js
var add = function add(x, y){
  return x + y;
};

console.log(add(1, 2)); // 3
```
이런 현상이 발생하는 이유는 JS엔진이 코드의 문맥에 따라 동일함 함수리터럴을 표현식이 아닌 문인 함수 선언문으로 해석하는 경우와 표현식인 문인 함수리터럴로 해석하는 경우가 있기 때문이다.  
함수이름이 있는 기명 함수 리터럴은 함수 선언문 또는 리터럴 표현식으로 해석될 가능성이 있다.  
기명함수 리터럴은 중의적 코드이므로 코드 문맥에 따라 해석이 달라질 수 있다.  
함수 이름이 있는 함수리터럴을 단독으로 사용하는 경우 함수 선언문으로 해석하고, 함수 리터럴이 값으로 평가되어야 하는 문맥에서는 함수 리터럴로 해석한다.  

함수가 생성되는것은 동일하나 함수를 생성하는 내부 동작에 차이가 존재한다.  
기명함수의 경우 함수 이름이 변수 처럼 사용가능한것처럼 보이지만 실제 함수 이름은 함수 몸체내에서만 호출 가능하다. (그렇다면 어떻게 부를수 있는것인가?)    
기명함수는 함수이름과 동일한 변수를 암묵적으로 생성해 함수를 할당한다.  

함수는 함수이름으로 호출하는것이 아니고 객체를 가리키는 식별자로 호출한다.  
JS엔진은 함수 선언문을 함수 표현식으로 변환 해 함수 객체를 생성한다고 볼 수 있다.  

#### 함수 표현식
JS의 함수는 함수는 값의 성질을 갖는 일급객체이다.  
일급 객체이기 떄문에 함수 리터럴로 생성한 함수 객체를 변수에 할당할 수 있다. (=함수 표현식)  
함수리터럴은 이름을 생략 가능 (=익명 함수)  
함수 표현식의 함수 리터럴은 이름을 생략하는것이 일반적이다.  
어차피 함수 이름은 함수 몸체 내부에서만 유효한 식별자이므로 함수이름으로 함수를 호출할 수 없다.  
```js
var foo = function add(x, y){
  return x + y;
};

console.log(foo(1, 2)); // 3
console.log(add(1, 2)); // ReferenceError: add is not defined
```

#### 함수 생성시점과 호이스팅
함수 선언문을 통해 정의한 함수는 함수 선언문 이전에 호출가능하다.  
함수표현식으로 정의한 함수는 함수 선언문 이전에 호출이 되지 않는다.  

함수 선언문으로 정의하면 런타임 이전에 함수객체가 먼저 생성됨 -> 함수 호이스팅  
변수 호이스팅의 경우 변수 선언이전에 참조를 하면undefined가 출력되지만 함수의 경우엔 호출이 가능하다.  
함수 표현식으로 함수를 정의 함수 호이스팅이 발생하는것이 아닌 변수 호이스팅이 발생한다.  
함수 호이스팅은 함수 호출 이전에 함수를 선언해야한다는 규칙을 무시하므로 함수 선언문 대신 함수 표현식을 사용할것을 권장한다.  

#### Function 생성자 함수
빌트인 함수 function생성자 함수에 매개변수 목록과 new 연산자와 함께 호출하면 함수 객체를 반환함 (new가 없어도 결과는 동일)  
함수 선언문이나 표현식과 다르게 동작하므로 function 생성자 함수를 만드는것은 바람직하지 않다.  

#### 화살표 함수
function 키워드 대신 화살표 =>를 사용하여 간략한 방법으로 함수를 생성한다.  
표현 뿐 아니라 내부동작 또한 간략화되었다.  

### 함수 호출
함수는 식별자와 한쌍의 소괄호인 함수 호출 연산자로 호출한다.  
호출연산자 안에는 0개 이상의 인수를 쉼표로 구분하여 나열하고, 함수가 호출되면 현재의 실행흐름을 호출된 함수로 옮긴다.  

#### 매개변수와 인수
함수를 실행하기 위해 필요한 값을 외부에서 함수 내부로 전달 할 필요가 있는경우 매개변수를 통해 인수를 전달하고 인수는 값으로 평가 될 수 있는 표현식이다.  
매개변수는 함수를 정의할때 선언, 함수 몸체에서 변수와 도일하게 취급한다.  
따라서 암묵적으로 변수가 생성되고 undefined로 초기화 된 이후 인수가 할당된다.  

매개변수의 스코프는 함수 내부이다.  
함수는 매개변수의 개수와 인수의 개수의 일치를 확인하지 않는다. 인수가 부족하면 부족한 매개변수의 값은 undefined로 초기화한다. 하지만 매개변수보다 인수가 많으면 초과분을 무시한다.

또한 동적 타입 언어이므로 JS는 함수를 정의할때 인수의 타입을 확인할 필요가 있다.  
내부에서 적절한 인수가 전달되었는지 확인하더라도 부적절한 호출을 방지할 수 없고 에러는 런타임에서 발생한다.  
따라서 타입스크립트와 같은 정적타입을 선언가능한 JS의 상위 확장을 도입해 호출을 방지하기도한다.

매개변수는 개수의 제한이 없다. 하지만 매개변수의 수가 늘어난다는것은 함수가 여러 일을 한다는것이다.  
이상적 함수는 한가지 일만하므로 아무리 많아도 매개변수가 3개를 넘지않는 것이 좋다.  

#### 반환문
함수는 return키워드와 표현식으로 이루어진 반환문을 통해 함수 실행 결과를 외부로 반환가능하다.  
함수 호출은 표현식이고 표현식은 값을 반환한다. 함수는 return키워드가 반환한 표현식의 평가 결과를 반환한다. 반환문은 함수 내부에서만 사용가하다.  
Node.js는 모듈시스템에 의해 파일별로 독립적 파일 스코프를 가지므로 파일의 가장 바깥에서도 반환문을 사용해도 에러가 발생하지 않는다.

#### 반환문의 역할
1. 함수 실행을 종료하고 함수 몸체를 나간다.
2. return 키워드의 뒤에 오는 평가식을 평가하여 반환한다. (명시적으로 지정하지 않으면 undefined반환)  

### 참조에 의한 전달과 외부 상태의 변경
인수로 객체 전달시 : 원본이 훼손되지 않는다.  
인수로 객체 전달시 : 원본이 훼손된다.  
  -> 이러한 문제 해결방법중 하나는 객체를 불변객체로 만들어 사용하는것이다.  
  객체의 상태 변경을 원천 봉쇄하고 객체의 상태변경이 필요한경우 객체의 방어적 복사를 통해 원본객체를 완전히 복제한다.  
깊은 복제를 통해 새로운 객체를 생성하고 재할당을 통해 교체하고 이런식으로 외부상태를 변경하지 않고 외부 상태에 의존하지 않는 함수를 순수함수라고 한다.  

### 다양한 함수의 형태
#### 즉시 실행 함수
함수정의와 동시에 즉시 호출되는 함수는 즉시 실행함수라고 한다.  
단 한번만 호출되며 다시 호출할 수 없고, 즉시 실행 함수는 이름이 없는 익명함수로 사용하는것이 일반적이다.  
즉시 실행 함수는 그룹연산자(...)로 묶어야한다. 함수 선언문 뒤의 ()가 호출연산자가 아닌 그룹연산자로 해석되기 때문이다.
```js
(function(){console.log('func')});
```

#### 재귀함수
재귀함수는 자기 자신을 호출하는 함수이다.  
재귀함수는 자기 자신을 무한 호출하므로 재귀를 멈출 수 있는 탈출 조건을 반드시 만들어야한다.  
탈출 조건이 없을 경우 함수가 무한 오출되어 스택 오버플로우 에러가 발생한다.  
재귀는 반복문 처리를 반복문없이 구현할 수 있다는 장점이 있지만 무한 반복에 빠질 위험이 있고, 이로인해 스택오버플로우 에어를 발생시킬수 있으므로 주의해서 사용해야한다.  

#### 중첩함수
함수 내부에서 정의된 함수를 중첩함수 또는 내부함수라고한다.  
중첩함수를 포함하는 함수는 외부함수라고 부른다.  

#### 콜백함수
함수의 매개변수를 통해 다른 함수의 내부로 전달되는 함수를 콜백함수라고 한다.  
매개 변수를 통해 함수의 외부에서 콜백함수를 전달받은 함수를 고차함수라고 한다.  
함수의 변하지 않는 공통 로직은 미리 정의해두고, 경우에 따라 변경되는 로직은 추상화해서 함수 외부에서 함수 내부로 전달한다.  
콜백함수는 중첩함수 처럼 고차함수에서 헬퍼 함수의 역할을 한다.  
고차함수는 콜백함수를 자신의 일부분으로 합성하고 매개변수를 통해 전달받은 콜백함수의 호출 시점을 결정해서 호출한다. 콜백함수는 고차함수에 의해 호출되며 이때 고차함수는 필요에 따라 콜백함수에서 인수를 전달할 수 있다.

콜백함수는 함수형 프로그래밍 패러다임뿐만 아니라 비동기 처리에도 활용되는 중요한 패턴이다.  
콜백함수는 비동기 처리 뿐 아니라 배열 고차함수에서도 사용한다.  

#### 순수함수와 비순수 함수
외부상태에 의존하지 않고 변경하지도 않는 부수효과가 없는 함수 : 순수함수  
외부상태에 의존하거나 외부상태를 변경하는 부수효과가 있는 함수 : 비순수 함수  

순수함수는 동일한 인수가 전달되면 동일한 값을 반환하는 함수이다.  
오직 매개변수만을 통해 함수 내부로 전달된 인수로에게만 의존해 반환값을 만들고 외부 상태를 변경하지 않는다.  

외부상태에 따라 반환값이 달라지는 (외부상태에 의존하는)함수를 비순수 함수라고한다. 비순수 함수의 특장은 함수 외부상태를 변경하는 부수효과가 있다는것이다.  
함수 내부에서 외부를 직접 참조하지 않더라도 매개변수로 객체를 전달받으면 비순수함수가 된다.
함수가 외부상태를 변경하면 상태변화의 추적이 힘들기 때문에 순수함수를 사용하는것이 좋다.  


#### 함수형 프로그래밍
순수 함수와 보조함수의 조합을 통해 외부상태를 변경하는 부수 효과를 최소화 하여 불변성을 지향하는 패러다임  
조건문이나 반복문은 로직의 흐름을 이해하기 어렵게 해 가독성을 해치고 변수의 값은 누군가에 의해 변경될 수 있으므로 오류 반복의 근복적 원인이 될 수 있다.  
JS는 멀티 패러다임 언어이므로 객체지향 프로그래밍 뿐 아니라 함수형 프로그래밍을 적극활용하고있다.

## 프로그래머스 2단계
### 괄호변환
알고리즘 스터디의 첫 문제  
입력되는 전체 문자열은 괄호의 균형을 보장한다. 이 문자열을 주어진 규칙에 맞추어 다시 만들어 반환하는 문제였는데 문제를 잘못읽어서 시간을 상당히 많이 소비했다.  
```js
// https://programmers.co.kr/learn/courses/30/lessons/60058
const recursive = function (p){
  // 재귀 종료 조건
  if(p === '') return '';
  // 사용할 함수를 초기화
  let left = 0; let right = 0; let i = 0;
  const arr = [];
  let fix = '';
  // 균형이 맞을때까지 반복
  while(1){
    // now는 현재 괄호
    const now = p[i++]; 
    // 여는 괄호면 left 증가 닫는 괄호면 right증가 
    now === '(' ? left++ : right++;
    arr.push(now);
    // 균형잡히면 반복문 탈출
    if(left === right) break;
  }

  // 균형이 깨졌다면 현재 문자를 놔두고 뒤쪽으로 진행 (재귀)
  if(arr[0] === ')'){
    fix += `(${recursive(p.slice(i))})`;
    // 방향 전환
    for(let j = 1; j < arr.length-1; j++){
      arr[j] === '(' ? fix += ')' : fix += '(';
    }
    return fix;
  }

  // 균형이 맞다면 arr을 문자열로 만들고 나머지를 재귀
  fix += arr.join('') + recursive(p.slice(i));
  return fix;
}
```
